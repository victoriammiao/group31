1. HTTP 状态码分类
HTTP 状态码分为几大类，每一类代表一种类型的响应：

2xx 系列（成功）：

200 OK：请求成功，通常用于 GET 请求。
201 Created：资源创建成功，通常用于 POST 请求。
204 No Content：请求成功但无返回内容，通常用于 DELETE 请求。
4xx 系列（客户端错误）：

400 Bad Request：客户端请求格式错误，例如请求参数不正确或缺少必需字段。
401 Unauthorized：用户未通过身份验证，通常用于请求需要身份验证的资源。
403 Forbidden：用户已经通过身份验证，但没有访问资源的权限。
404 Not Found：请求的资源不存在，例如请求的用户 ID 不存在。
409 Conflict：请求的资源冲突，通常用于资源唯一性冲突的情况（例如用户电子邮箱已存在）。
5xx 系列（服务器错误）：

500 Internal Server Error：服务器内部错误，通常用于不可预见的错误。
502 Bad Gateway 和 503 Service Unavailable：服务器在处理请求时出现了临时的错误。
